# 运行时记忆与检索接地:三问题框架的本仓库落地

> 来源:三个根本问题(知识固化/幻觉/过早收敛)共享同一根源——当前大模型把
> 「知识存储、推理、表达」全部压缩在一套静态权重里。工程方向是把单一模型拆成
> 系统:冻结的核心 + 可更新的外部记忆 + 独立的验证器 + 显式的探索阶段。
> 本文档记录该框架,并标注其中哪些已在本仓库落成可运行代码。文献见 REFERENCES §24。

## 1. 知识固化(持续学习而不遗忘)

难点是灾难性遗忘:学新知识会覆盖旧权重(EWC 论文给出量化机制)。方案谱系:

| 方案 | 机制 | 本仓库状态 |
|---|---|---|
| 外挂记忆 (RAG) | 新知识入可检索外部库,不动权重 | **已落地**:`POST /v1/vector/memory` 运行时写入,增量编码入 SQLite 向量库,`kind="memory"`,重建索引时保留 |
| 参数高效微调 (LoRA) | 少量附加参数,插件式装卸 | 不适用(本仓库不持有权重);其"可装卸"语义由 `DELETE /v1/vector/memory/{id}` 对应实现 |
| 回放机制 | 训练新知识时混入旧数据 | 不适用(无训练侧) |
| EWC / 模块化网络 | 重要突触保护 | 研究前沿,仅作记录 |

落地选择依据:外挂记忆是唯一不需要权重访问的方案——"给 AI 配笔记本而不是做
脑外科手术"。MemGPT 的分层记忆(主上下文/外部存储)是同方向的工程化参照。

## 2. 解决幻觉

幻觉本质:模型在「补全最可能的文本」而非「陈述已验证的事实」,无内置真值检查器。
缓解手段分层(根治尚无解,因为生成与验证在当前架构里是同一过程):

| 层 | 手段 | 本仓库状态 |
|---|---|---|
| 检索接地 (Grounding) | 回答必须引用检索来源,无来源则说不知道 | **已落地**:`POST /v1/agents/collaborate` 带 `ground=true` 时,先检索本地向量库(skills/knowledge/memory),把带编号来源前置进任务,并指示"资料不足时明说不知道";低于相似度阈值不接地(避免在噪声上接地) |
| 不确定性表达 | 校准置信度,"我不确定"是合法输出 | 文档级:deepdive 证据分级与置信度校准实践 |
| 自我验证 | 另一模型核查 / self-consistency | 部分落地:peer_review 互评即跨模型核查;采样一致性未做 |
| 工具调用 | 不该靠记忆的事外包出去 | pipeline 工具链已具备 |

与 deepdive §4.3 验证器隔离一致:核查者与生成者分离(跨提供商路由)。

## 3. 过早收敛(探索不足)

| 手段 | 本仓库状态 |
|---|---|
| 温度/采样策略发散 | committee/peer_review 多提案并行即推理时多路径(ToT 的扁平变体) |
| 先生成多候选再评判 | **已落地**:peer_review 把「生成」与「评判」拆为两阶段,评判用 temperature=0 |
| 训练层熵奖励/课程学习 | 不适用(无训练侧) |

## 4. 系统化总结

三问题的共同解是「拆成系统」,本仓库的对应物:

- 冻结的核心 = 上游模型(只调用,不微调);
- 可更新的外部记忆 = 向量库 `memory` kind(写入/列出/删除,重建幸存);
- 独立的验证器 = peer_review 跨提供商互评 + 围栏防注入;
- 显式的探索阶段 = 多提案并行后再收敛评判。

边界与校准:接地只能降低无来源断言的频率,不能保证引用被正确使用(模型可能
引用了来源却曲解内容);记忆写入无去重与冲突检测,矛盾笔记会同时被检回——这
两条是后续轮次的真实缺口。
