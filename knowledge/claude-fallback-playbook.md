# 安全分类器与回退:检测、归因与应用层照搬指南

> 性质:机制深析 + 可操作 playbook。对应 `model-understanding-roadmap.md` 第 3 条(A3)。
> 事实出处:Fable 5 发布公告(机制概述见 `claude-fable5-opus48.md` §5);
> 本文的检测/归因细则为工程推导,标注【推断】处为合理外推。时点:2026-06-10。

## 1. 机制再拆解:三个组件,三种失效

公开机制 = **独立分类器(守门员) + 主模型(Fable 5/Mythos 5) + 回退体(Opus 4.8)**。
每个组件有自己的失效面:

| 组件 | 失效模式 | 对调用方的表现 |
|---|---|---|
| 分类器 | 误伤(false positive,设计上偏保守) | 合法请求被回退,质量/风格突变 |
| 分类器 | 漏放(false negative) | 对调用方无感,属厂商风险面 |
| 回退体 | 能力差额 | <5% 会话里实际能力是 Opus 4.8 档 |

关键认知:**误伤是设计预期而非 bug**;受保护类别(网络安全、生物/化学、蒸馏)的
*邻域*合法请求——安全研究、漏洞修复说明、模型压缩论文复现、生化课程内容——都在误伤半径内。

## 2. "本次响应是否被回退"检测清单

按成本从低到高依次检查:

1. **显式信号**:响应中的告知字段/文案(官方承诺"会告知用户")——解析并落日志,这是第一信源;
2. **用量指纹【推断】**:同类请求下思考 token 量、输出风格突变(Fable 5 强制 adaptive thinking,
   回退后思考形态可能不同);单独不可靠,只做旁证;
3. **质量指纹【推断】**:你的评测集里"只有 Fable 5 能过"的样本突然失败;
4. **聚合层面**:回退率(fallback_rate)按 prompt 模板分组统计——某模板回退率显著高于基线,
   说明该模板措辞踩中了分类器的敏感邻域。

落地要求(最低限度):每条响应日志必须有 `requested_model / answered_model / fallback: bool` 三个字段。
没有这三个字段,§4 的归因流程无法执行。

## 3. 被误伤时的处理决策流

```
请求被回退(且你判断属合法用途)
├─ 该任务 Opus 4.8 也能胜任?
│   ├─ 是 → 接受回退,什么都不做(这正是"可接受的回退体"设计的本意)
│   └─ 否 → 改写表述后重试一次:
│       ├─ 去除受保护类别的触发性措辞(exploit/合成路线/蒸馏窃取式表述),
│       │   改为中性工程语言;只改措辞,不改任务实质
│       ├─ 改写后通过 → 记录"措辞敏感"样本,沉淀进 prompt 规范
│       └─ 仍被回退 → 停止重试(分类器不会被重试绕过),走人工通道/厂商支持
└─ 重要红线:不得通过逐步改写"试探"分类器边界来运送实质违规内容——
    那是越狱行为,且会污染你账号的风险画像【推断:厂商有账号级风控】
```

## 4. 质量回归的归因顺序(回退感知版)

线上质量抖动时,按此顺序排查,**每步排除后才进下一步**:

1. 查 `fallback` 字段:抖动样本是否集中在被回退请求 → 是,则是回退问题,按 §3 处理;
2. 查缓存:prompt caching 命中段是否含已过期的旧版本内容;
3. 查模型版本:上游是否静默更新(对照评测集重跑基线);
4. 查自己:prompt/上下文组装是否有变更(diff 部署记录);
5. 都排除 → 扩充评测集复现,而不是盲改 prompt。

## 5. SLI 与告警细则

| 指标 | 定义 | 告警阈值建议 |
|---|---|---|
| fallback_rate | 被回退请求数 / Fable 5 请求数 | 7 日均值 > 5%(偏离官方基线)或单模板周环比翻倍 |
| fallback_quality_gap | 回退请求与未回退请求的评测分差 | 差值扩大趋势(说明你把不该给 Fable 5 的任务路由错了,或回退体不胜任) |
| rewrite_success_rate | 措辞改写后通过的比例 | < 50% 时说明触发的是任务实质而非措辞,改写策略失效 |

## 6. 把"守门员模式"照搬进应用层

这个结构可在应用自身的敏感能力分级中复用:

- **结构**:强能力路径(如自动执行代码/动钱操作)前置一个**独立**判定器,
  命中即降级到受限路径(只读/需人工确认),并明确告知用户发生了降级;
- 照搬细则:
  1. 判定器必须独立于主路径(独立模型/规则引擎),否则被同一个 prompt 注入一起攻破;
  2. 回退体必须"可用且无害",而不是直接拒绝——保住可用性是这个范式优于拒答的全部理由;
  3. 偏保守起步 + 用误伤日志迭代降误报,顺序不能反;
  4. 降级事件是一等公民指标(对应 §5),不是 debug 日志。
- 反模式:把判定逻辑写进主模型的 system prompt 里("如果用户请求 X 则拒绝")——
  这不是守门员,这是把门和房子焊在一起。

## 诚实边界

- 分类器的具体实现、阈值、类别全集均未公开;§2 的指纹类检测手段全部是【推断】,
  唯一可靠信号是官方的显式告知字段。
- ">95% 会话不触发"是发布时点的官方数字,会随护栏迭代变化;阈值建议应随之重标。
- §6 是结构类比,不是安全审计建议;高危场景(资金、生产系统写操作)需要独立安全评审。
