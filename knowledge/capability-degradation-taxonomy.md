## D1 上游模型版本更新

- **症状**:未做任何变更,质量/风格/成本突变;此前稳定的边缘行为(格式遵循、拒答边界)翻转。
- **检测**:版本号入日志(响应头/字段);新版本公告触发评测集全量重跑(基线对比)。
- **第一处置**:若供应商支持版本钉住(pinning),钉住旧版,争取迁移窗口。
- **根治**:迁移流程化(roadmap 第 8 条):回归门禁 → 双跑对比期 → 切流 → 回滚条件。
  prompt 不要依赖未文档化的行为(对空白容忍、隐式格式假设)——那些是版本间最先碎的。

## D2 安全回退

- 全部细则见 `claude-fallback-playbook.md`;在本分类中的定位:**唯一有官方显式信号的退化源**,
  因此应当第一个排查(归因顺序的第一步,排除成本最低)。

## D3 prompt 漂移(补丁式增长)

- **症状**:prompt 越改越长,新指令与旧指令隐性矛盾;模型"选择性服从",同 prompt 行为方差变大。
- **检测**:prompt 版本化 + 每版过门禁;定期跑"指令一致性审计"(让模型列出 prompt 中互相冲突的指令——
  模型找矛盾的能力远强于人眼)。
- **第一处置**:回滚到最后一个过门版本。
- **根治**:`capability-fixation.md` §6 的分层存放——纪律进 skill、事实进知识库、
  每条新增指令必须声明"它修的是哪个评测样本",修不出样本的指令不许进 prompt(对应棘爪流程)。

## D4 上下文腐烂

- **症状**:长会话/长上下文中后段质量下降;注入材料多了反而答得差("lost in the middle"类衰减)。
- **检测**:按"上下文长度分桶"统计质量分——质量随长度单调下降即确诊;
  对比"只注入 top-k 相关材料"与"全量注入"的 A/B。
- **第一处置**:收紧注入预算(本仓库 ranker 的 token 预算机制即此),宁缺毋滥。
- **根治**:检索精度优先于召回量;长任务改用记忆位(笔记式持久化)替代"全历史塞上下文"——
  Fable 5 的记忆收益证据(3×)说明"会用记忆"优于"上下文更长"。

## D5 缓存陈旧

- **症状**:更新了 system prompt/skill 后部分流量仍按旧行为响应;质量抖动与缓存命中相关。
- **检测**:缓存命中标志入日志;发布后对比命中/未命中流量的行为一致性。
- **第一处置**:发布时主动失效相关缓存段。
- **根治**:缓存键包含 prompt 版本号——版本变更自动失效,杜绝人工遗漏。

## D6 评测过拟合(古德哈特)

- **症状**:评测分持续上升,线上投诉/抽检质量持平或下降;prompt 里出现针对具体评测样本的特判。
- **检测**:封存集分数与迭代集分数的差距趋势——差距持续拉大即确诊。
- **第一处置**:立刻停止以当前迭代集为目标的优化。
- **根治**:封存集纪律(信息预算记账+轮换,细则见 `claude-eval-methodology.md` §5)
  + 评测集持续注入生产新样本(同文 §1.2/§4);本仓库的确定性封存切分已落码于 `eval_store.is_sealed`。
  这是八种退化源里唯一"仪表盘越好看越危险"的,必须靠流程而非警觉防御。

## D7 任务分布漂移

- **症状**:总体指标平稳但某新兴任务类型质量差;路由/effort 标定对新流量失效。
- **检测**:按任务类型分层监控占比与质量;新类型占比 >10% 触发评测集重建(已约定于评测生命周期)。
- **第一处置**:新类型临时路由到最强配置(Fable 5 + xhigh)——先保质量再调成本。
- **根治**:评测集季度轮换;路由阈值随每次重扫重标。

## D8 检索/记忆位污染

- **症状**:模型引用了过期/错误的检索内容且无法自我察觉;同一错误跨会话复现(已被记忆固化)。
- **检测**:对带引用的产出抽样核源;记忆位内容定期审计(时点标注过期的条目)。
- **第一处置**:隔离污染条目(下线该 chunk/笔记),而不是改 prompt 对抗它。
- **根治**:进入检索库/记忆位的内容一律带时点与出处(本知识库的"诚实边界"惯例即此);
  写入记忆位的内容过一道轻量校验(出处可溯 + 与现有条目无冲突)。

## 使用方式:归因总顺序

线上质量回归时,按隐蔽性从低到高排查(便宜的先查):
**D2 回退 → D5 缓存 → D1 版本 → D4 上下文 → D3/D8 语料 → D7 分布 → D6 评测自身**。
前三步有显式信号,五分钟可排除;后五步需要分层统计与审计。
查到任何一步确诊,处置后必须走棘爪流程(样本入评测集),否则同一退化会换个形态回来。

## 诚实边界

- 八分类是工程归纳,不保证完备;新的上游机制(如更复杂的路由式服务)会引入新退化源,
  发现一种补一种。
- D4 的注意力衰减实证在不同模型代际间差异大,"长度分桶检测"的结论只对当前模型版本有效。
- 各"根治"手段的成本差异巨大,排序时已偏向"流程化优于人工警觉"的取向——这是立场,不是定理。
