# 可证伪预测跟踪表

## 回填约定

- **触发条件**:任一前沿厂商(Anthropic / OpenAI / Google / DeepSeek / Meta·开源旗舰)发布大版本,
  或 METR 发布新的任务时长测量 → 7 日内回填。
- 每次回填:在对应预测的证据栏追加一行 `日期 | 事件 | 支持/反对/中性 | 一句话理由`,
  并更新判定栏。证据只许追加,不许改写历史行。
- **判定规则**:到检验日期时,按证据栏多数倾向给出 `成立 / 部分成立 / 不成立 / 证据不足`;
  任何"不成立"必须回写 `model-capability-evolution.md` 修正对应论点,这是建表的全部目的。

## 判据可执行度审计(第 38 轮新增,源自预测 6 的碰撞结论)

每条预测按"检验时刻判据能否机器/流程化裁决"评级,低执行度预测在此补强:

| 预测 | 执行度 | 补强 |
|---|---|---|
| 1 | 中 | "明显大于"无量化——补:以 SWE-bench 类基准年增幅 vs 创意写作类人评年增幅之差 >10pp 为"明显" |
| 2 | 中高 | METR 翻倍时间序列是公开数字,"不放缓"=下一个翻倍周期 ≤ 上一个 × 1.3 |
| 3 | 高 | 基线数字已记录(3×),同 harness 复测即得 |
| 4 | 低 | "发布滞后加大"难观测(内部能力非公开)——降级为方向性预测,只记双形态发布的采用计数 |
| 5 | 低 | 触发型条款,本就不要求可执行;触发后第一问固定为"反馈信号来源是什么" |
| 6 | 高 | 已升级为实验协议(见下) |

纪律:回填证据时若发现判据仍裁决不了,先修判据(记录修订理由)再判,不许"凭感觉倾向"。

- **检验日期**:下一次前沿大版本发布时点 + METR 后续报告。
- **当前判定**:证据不足。
- 证据:
  - 2026-06-09 | Fable 5 官方表述"任务越长越复杂,领先越大" | 支持 | 厂商已主动采用该叙事

## 预测 3:记忆管理被显式训练

- **判据**:同 harness 下,新模型相对上代的记忆收益继续拉大(基线:Fable 5 vs Opus 4.8 的 3 倍)。
- **检验日期**:下一代发布时点。
- **当前判定**:证据不足(基线已记录)。

## 预测 4:安全决定发布节奏

## 预测 6:无棘轮系统在模型迁移时退化(来自 capability-fixation.md)

- **判据**:本仓库下一次更换/升级底层模型时,若产出质量评测集(路线图 8.6)尚未建立,
  将至少出现一次"无法在 24h 内归因"的产出质量回归;若评测集已建立,则迁移可在
  门禁报告中量化通过或阻断。这是对棘轮理论的自指检验。
- **检验日期**:下一次本仓库底层模型迁移时点。
- **当前判定**:未触发(2026-06-10 立项,基线:代码测试 327 个,产出门禁 0 个)。
- **实验设计已可执行(2026-06-10 第 34 轮碰撞更新)**:本预测从"等事件发生"升级为
  "有协议的实验"——迁移时按 `model-migration-checklist.md` 逐项执行,每一步留痕:
  - 归因耗时的计时起点 = D1 检测器报警或公告时点(`degradation.detect_model_change` 给出
    run_id 与 created_at,不再依赖事后回忆);
  - "门禁量化通过/阻断" = `scripts/eval_gate.py` 的 verdict 字段,exit code 即结论;
  - 工具基线更新:代码测试 367 个;门禁机制已落码但真实 regression 样本仍为 0——
    因此预测的"无评测集分支"目前仍是默认路径,这正是 8.6 剩余工作的紧迫性所在。
  碰撞结论:可证伪预测的价值随"判据的机器可执行度"上升;判据不可执行的预测
  会在检验时刻被解释空间淹没(事后怎么说都行)。本表其余预测应逐条向此标准看齐。
- 证据:(暂无)
